# MariaDB 
## Tanım ve Kullanım Amacı  
MariaDB, **MySQL’in (1995 çıkışlı) özgür bir çatalı** olarak Michael “Monty” Widenius önderliğinde geliştirilmiş açık kaynaklı bir ilişkisel veritabanı yönetim sistemidir. Oracle’ın MySQL’i 2009’da satın alması sonrası orijinal geliştiriciler MySQL’e alternatif olarak MariaDB projesini başlatmıştır. GPL lisanslıdır ve topluluk tarafından geliştirilmeye devam etmektedir. MySQL ile yüksek düzeyde uyumluluğu sayesinde (`drop-in replacement`), MySQL kullanan uygulamaların çoğu MariaDB’de de sorunsuz çalışır. Dünyada Wikipedia, WordPress.com ve Google gibi büyük sistemler de MariaDB kullanmaktadır. Genel kullanım alanları:
- **Web uygulamaları:** WordPress, Drupal, Joomla gibi CMS’ler için veri deposu.  
- **Veri analitiği:** OLAP sorguları, iş zekası için.  
- **Dağıtık/çoklu sunucu yapıları:** Galera Cluster veya replication ile yüksek erişilebilirlikli sistemler.  
- **Gömülü veritabanı:** Hafif, entegre çözümlerde de kullanılabilir.  

Şirket tarafından açık kaynak versiyonu (MariaDB Community Server) dışında [kurumsal sürüm](https://mariadb.com/products/) da sağlanmaktadır.  

## Tarihçe ve Sürümler  
MariaDB’nin ilk kararlı sürümü Ocak 2009’da yayımlandı. 5.5’e kadar versiyon numaraları MySQL ile paralel giderken, MySQL 5.6’ya geçişle MariaDB “10.0” seri numarasıyla bağımsız bir sürüm çizgisine geçti. Şu an (2026 itibarıyla) MariaDB 10.x sürümleri güncel olup, geliştiriciler yeni özellikleri Oracle MySQL’le paralel olmadan özgürce ekleyebilmektedir. Örneğin veri şifreleme, JSON destekleri, yeni depolama motorları (Aria, MyRocks, etc.) MariaDB’de erken sunulan özelliklerdendir.  

## Mimarisi ve Temel Bileşenler  
MariaDB, MySQL’den farklı olarak **modüler mimari** üstüne kuruludur. En önemli unsurlar:  
- **Storage Engine (Depolama Motorları):** Her tablo için seçilebilen farklı motorlar vardır. Varsayılan olarak **InnoDB** kullanılır, bu motor ACID uyumlu, kilitlenen (lock) güncelleme ve **yabancı anahtar** desteği sunar. MariaDB ayrıca MyISAM uyumlu, ancak Aria adıyla geliştirilmiş bir motor sunar. Aria, MyISAM’in yerine gelebilecek hızlı bir seçenek olup yüksek oranda veri kopyalamaya uygun bir altyapı sunar. Üçüncü parti depolama motorlarıyla (MyRocks, Cassandra Connector vb.) genişletilebilir.  
- **SQL İşleyici (Mikro Çekirdek):** SQL sorguları alır, sözdizimini çözer, sorgu planı oluşturur. Sorgu optimize edici (optimizer) karmaşık sorgularda en hızlı yolu belirler. MariaDB’nin optimizer aracı gelişmiş izleme günlükleri sunar (ör. optimizer trace).  
- **Bellek Havuzu (Buffer Pool):** InnoDB’in veritabanı sayfalarını ve indekslerini tutan bellek bölgesi. Büyük hafıza ayırarak disk erişimini azaltabilirsiniz.  
- **Günlükler (Redo/Undo, Binary Log):** Veritabanı değişiklikleri için InnoDB redo/undo logları ve MySQL uyumlu binlog (ikincil sunucu replikasyonu için) bulunur.  
- **Sunucu Daemon’ı (mysqld):** Bağlantıları dinler, gelen talepleri işleyen merkezi süreç. 

Özetle, MariaDB tıpkı MySQL gibi **çok depolama motoru** destekleyen bir sunucudur. Bir tablo yaratırken `ENGINE=InnoDB` veya `ENGINE=Aria` gibi komutlarla motor seçilebilir. Standart kurulumda InnoDB aktiftir, Aria sistem tabloları ve geçici tablolar için kullanılır.

## Kurulum ve Yapılandırma Örnekleri  

**Ubuntu/Debian (APT):**  
```bash
sudo apt update
sudo apt install mariadb-server mariadb-client
```
komutları MariaDB’yi yükler. Kurulum sonrası `sudo mysql_secure_installation` komutuyla root parolası belirlenmeli, gereksiz örnek tablolar silinmelidir. Hizmet normalde otomatik başlar; `sudo systemctl status mariadb` ile çalışıp çalışmadığını kontrol edin.

**CentOS/RHEL (YUM/DNF):**  
CentOS7/RHEL7 paket yönetiminde MariaDB 5.5 bulunur; daha yeni sürümler için MariaDB deposu kurulabilir. Örneğin:  
```bash
sudo yum install mariadb-server
```
komutu MariaDB sunucusunu yükler. CentOS8 ve üstü için `dnf` kullanılabilir. Yüklemenin ardından:  
```bash
sudo systemctl start mariadb
sudo systemctl enable mariadb
```
ile servis başlatılır. Ardından `mysql_secure_installation` çalıştırılarak güvenlik ayarları yapılmalıdır.

**Veri Tabanı Örneği (WordPress):** Yukarıda bahsedilen SQL örneğinde görüldüğü gibi bir WordPress veritabanı kurulumu şudur:  
```sql
CREATE DATABASE wordpress DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
GRANT ALL ON wordpress.* TO 'wordpress_user'@'localhost' IDENTIFIED BY 'sifre123';
FLUSH PRIVILEGES;
```
Bu şekilde WordPress için özel bir veritabanı ve kullanıcı tanımlanır.  

## MariaDB + WordPress + Nginx Etkileşimi  
Nginx ve WordPress kullanırken backend veri katmanı MariaDB’dir. Yukarıda (WordPress bölümünde) anlatıldığı üzere WordPress, sitenin içerik verilerini MariaDB’den çeker. MariaDB, WordPress’in gereksinimlerini karşılayacak biçimde ACID uyumlu InnoDB depolama motorunu standart olarak kullanır. Nginx sadece istemcilerle HTTP üzerinden iletişim kurar; MariaDB’ye bağlanma işlemi doğrudan PHP üzerinden yapılır (dolayısıyla Nginx-MariaDB arası istemci-sunucu ilişkisi yoktur). Sonuç olarak, WordPress’in veri işlemleri MariaDB tarafından yürütülürken Nginx web sunucusu ön yüzde yüksek performans sağlar. Özellikle çok sayıda okuyucu (frontend) ise Nginx yönetimi, veri işlemleri MariaDB tarafından arka planda etkin şekilde yapılır.  

## Güvenlik, Performans ve Bakım İpuçları  
- **Root Parolasını Tanımlama:** Kurulum sonrası `mysql_secure_installation` komutunu çalıştırmak çok önemlidir. Root (idari) kullanıcı parolasını belirleyin, anonim kullanıcıları silin, uzak root erişimini devre dışı bırakın, test veritabanını kaldırın.  
- **Bağlantı Noktası ve Güvenlik:** Varsayılan port 3306’dır. Uzaktan erişim gerekiyorsa firewall (iptables, firewalld) üzerinden yalnızca güvenilen IP’lere izin verin. Aksi takdirde MariaDB sunucusu sadece localhost’a bağlansın.  
- **SSL/TLS:** Mümkünse istemci-sunucu bağlantısını SSL şifrelemesiyle güvence altına alın. MariaDB 10+ sürümleri TLS/SSL desteği içerir. Sertifika kurulumu ile bağlantılar gizli tutulur.  
- **Veri Şifreleme:** Son sürümlerde “data-at-rest encryption” (disk üzerinde şifreleme) kullanılabilir. Önemli veriler için InnoDB tablespace şifreleme ayarlanabilir.  
- **Performans Ayarları:** `/etc/mysql/mariadb.conf` veya `my.cnf` dosyasında **innodb_buffer_pool_size** bellek havuzu (örneğin RAM’in %70-80’i), **query_cache_size** (eski sürümler için), **tmp_table_size** gibi ayarları mevcut donanıma göre optimize edin. Sürekli yavaş sorgular varsa loglayıp indeks ekleyin.  
- **Replikasyon/Küme:** Büyük ölçeklerde Galera Cluster veya replikasyon kurarak yüksek erişilebilirlik ve yük paylaşımı sağlayabilirsiniz. MariaDB bu konuda dahili destekler sunar.  
- **Bakım ve Yedek:** Düzenli olarak `mysqldump` veya `mariabackup` ile veritabanı yedeği alın. Periyodik olarak `OPTIMIZE TABLE` komutuyla tablolardaki boş alanı temizleyin. İzleme için `SHOW PROCESSLIST;` ve `INFORMATION_SCHEMA` tabloları kullanılabilir.  

## Yaygın Sorunlar ve Çözümler  
- **Yetkilendirme Hataları:** WordPress’in veritabanına bağlanamaması genelde yanlış kullanıcı/parola ya da yetkisizlikten kaynaklanır. `GRANT` komutlarını doğru çalıştırdığınızdan ve parolayı `wp-config.php` ile eşleştiğinden emin olun.  
- **Karakter Seti Sorunları:** Türkçe karakter hatalarına karşı veritabanı `utf8mb4` kullanılması tavsiye edilir. Örneğin: `CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci`. Ayrıca `wp-config.php` içinde `define('DB_CHARSET', 'utf8mb4');` ayarlayabilirsiniz.  
- **Bağlantı (Bind) Ayarları:** Eğer uzak bir host üzerinden bağlanacaksanız `/etc/mysql/mariadb.conf` veya `/etc/my.cnf` dosyasında `bind-address = 0.0.0.0` olarak ayarlanmalı ve gerekli port açılmalıdır.  
- **Veri Yedek Hataları:** `mysqldump` esnasında büyük tablolar veya işlem fazlalığı varsa dump başarısız olabilir. `--single-transaction` gibi parametrelerle kesintisiz yedek alma veya `mariabackup` kullanabilirsiniz.  
- **Sunucu Çökmesi:** Crash veya çökme durumunda veritabanının tutarsızlık göstermemesi için InnoDB’den yararlanın. `innodb_force_recovery` modlarıyla acil durum kurtarma yapılabilir.
